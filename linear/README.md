# Linear Rotation-Accumulation Experiment

## Overview

A novel toy task testing Transformer's **long-range dependency** and **order discovery** capabilities (without positional encoding) using orthogonal matrix dynamics.

## Mathematical Formulation

### Dynamical System

The sequence is generated by a linear dynamical system with orthogonal transitions:

```
h_0 = one_hot(start_token)
h_{t+1} = R @ h_t + one_hot(x_t)
x_{t+1} = argmax(h_{t+1})
```

Where:
- **R**: Fixed orthogonal matrix (V × V) - defines the physics of the system
- **h_t**: Hidden state at time t (maintains cumulative history)
- **x_t**: Token at time t (one-hot encoded)
- **argmax**: Deterministic next-token selection

### Key Properties

1. **Orthogonality**: `R^T @ R = I` ensures:
   - State norm preservation: `||R @ h|| = ||h||`
   - No gradient vanishing/exploding
   - Rich spectral structure (eigenvalues on unit circle)

2. **Long-Range Dependencies**:
   - History propagates through powers of R: `h_t ~ R^t @ h_0 + R^{t-1} @ x_0 + ... + x_{t-1}`
   - All past tokens contribute to current state
   - Changing order → completely different trajectory

3. **Deterministic Ground Truth**:
   - R matrix defines the "physics" of the system
   - Given bag and start token, sequence is (ideally) uniquely determined
   - No ambiguity in what the model should learn

## Why This Task?

### Problems with Previous Experiments

1. **Lossy Copy**: `x1 = x0 // 2` - lossy rule, limited depth
2. **Causal Chain**: Only 2-3 layers of dependencies
3. **DAG**: Complex but lacks clear ground truth

### Advantages of Linear Rotation

- ✅ **Clear Ground Truth**: R is the rule, no ambiguity
- ✅ **True Long-Range**: All history matters, not just k-step dependencies
- ✅ **Order Sensitive**: Permuting tokens changes output completely
- ✅ **Mathematically Principled**: Grounded in linear dynamical systems theory

## Parameter Guidelines

### Default Parameters (Start Here)

```python
vocab_size = 16      # V
seq_length = 20      # L
hidden_dim = 16      # D = V (simplifies implementation)
ortho_mode = 'random'  # Haar measure orthogonal matrix
```

### Tuning If Uniqueness < 80%

1. **Increase vocab_size**: 16 → 32 → 64
   - More tokens → less collision probability
   - Trade-off: Larger model needed

2. **Switch to permutation**: `ortho_mode='permutation'`
   - Sparser R matrix
   - May have higher uniqueness but simpler dynamics

3. **Reduce sequence length**: L=20 → L=15
   - Shorter sequences → less collision chance
   - Trade-off: Easier task, less interesting

### Tuning If Margin Too Small/Large

- **Margin < 0.1** (task too hard):
  - Use `mode='argmax'` (no temperature)
  - Consider normalizing h_t
  - Reduce sequence length

- **Margin > 5.0** (task too easy):
  - Increase sequence length
  - Add noise to dynamics

## Technical Details

### Uniqueness Check (Local Greedy)

The `verify_uniqueness()` method performs a **local greedy check**:
- At each step t, check if any other unused token in bag also satisfies argmax
- This is conservative: if local check passes, global uniqueness guaranteed
- No need for expensive DFS/backtracking

### Orthogonal Matrix Generation

```python
# Random orthogonal (Haar measure)
from scipy.stats import ortho_group
R = ortho_group.rvs(dim=vocab_size)

# Permutation matrix (sparse)
perm = np.random.permutation(vocab_size)
R = np.eye(vocab_size)[perm]
```

### Initial State

- `h_0 = one_hot(tokens[0])` - start token initializes system
- tokens[0] is given (part of input)
- Model task: Predict tokens[1], tokens[2], ..., tokens[L-1]
